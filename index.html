Things Mozillians do


  • Firefox, the web browser!
  • A bunch of building machines (in the cloud and bare metal), signing servers, testing servers, ...
  • Encrypted storage of user's history, passwords, etc. via Firefox Accounts.
  • Thimble, Internet Health Report, Rust, Servo, WebVR, add-ons, it goes on and on.


  • Most things we do are open, public, and we invite everyone to participate. Even for security!
  • 1200-300 people world-wide, remote and in offices
  • ~50 security engineers in different teams
  • Product Security, Services Security, Enterprise Information Security (that's us!), ...

Different security teams


  • Product security (they write, review Firefox code)
  • Service security (they secure specific web services)
  • Enterprise Information Security (we do incident response, risk analysis, pen-tests, ...)
  • Security is also everyone's job

What does a security incident look like?


  • Most common: credential leaks ("oops published this online"), phishing
  • Most dangerous: data leaked/modified through compromise
_

Incident *Response*


  • Powerful tooling
    • We write tools that capture program execution
    • We write tools that let us query system state ("did you see this IP address?") simultaneously
    • We look at network packets
    • We automatically log, alert, and make data available ("big data")
  • Speed and agility
  • Plan ahead!
  • Stay informed

Challenges in day to day work


  • Security is a process
    • Assess risk, re-assess risk...
    • Integrate checks in the life-cycle
    • You're never done
  • Risk perception does not come naturally
    • Are you afraid to travel in a plane?
    • It's about how bad it can get, when it'll happen (not if)
_

Some security principles


  • Be curious!
  • Keep it simple ... safe ("KISS")
  • Always think in terms of: data in, data out
  • Compartimentalize data, etc. and only grant access on a need-to-know basis
